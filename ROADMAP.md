# 요트 다이스 멀티플레이 및 데이터 연동 로드맵

현재 증강 요트 다이스 프로젝트에서 앞으로 구현해야 할 3가지 핵심 기능(로비 플레이, 게임 로그, 게임 결과 DB 연동)에 대한 마스터 계획서입니다. 에이전트들은 작업 시작 전 이 문서를 참고하여 구조적 방향성을 잃지 않도록 합니다.

## 1. 요트 다이스 로비 플레이 구현 (확장성 고려)
증강 요트 다이스로 확장 가능하도록 클래스 기반/모듈화된 아키텍처를 설계합니다.
- **방 생성 및 입장 (Lobby System)**:
  - Firestore의 ooms 컬렉션을 활용하여 방장(Host)이 방을 생성하고, 6자리 입장 코드를 통해 클라이언트(Guest)가 접속.
  - 플레이어 준비(Ready) 상태 동기화 및 게임 시작 트리거 로직 구현.
- **게임 상태 동기화 (State Sync)**:
  - 주사위 굴림, 카테고리 락인(점수 선택) 등 각 턴의 상태를 ooms 문서 내 gameState 필드를 통해 양방향 실시간 리스너(onSnapshot)로 동기화.
  - **확장성**: 일반 요트 다이스 로직(YachtLogic)을 기본 베이스로 두고, 추후 AugmentedYachtLogic이 이를 상속/확장하여 사용할 수 있도록 구조 분리.

## 2. 게임 플레이 로그 시스템 구현
진행 상황을 텍스트로 기록하여 양측 플레이어가 상대방의 행동을 파악할 수 있도록 합니다.
- **로그 매니저(LogManager.js)**:
  - [System] P1이 방에 입장했습니다., [Action] P2가 주사위를 굴렸습니다. (남은 굴림: 2), [Score] P1이 Full House에 25점을 기록했습니다. 와 같은 포맷으로 로그 생성.
- **UI 연동**:
  - game.html 내에 스크롤이 가능한 투명/반투명 로그 컨테이너 추가.
  - 새 로그 추가 시 자동으로 스크롤이 맨 아래로 내려오도록 처리.
- **오픈 퀘스천 (UI 위치)**:
  - 주사위 보드 우측 빈 공간, 점수판 하단, 혹은 별도의 토글형 모달 중 사용자 피드백에 따라 확정 예정.

## 3. 게임 데이터 DB 저장 기능 (매치 통계)
플레이어의 전적을 기록하여 랭킹 및 프로필에 반영합니다.
- **Firestore 연동 (matchHistory)**:
  - 게임 종료 시 방장(Host) 클라이언트가 최종 점수표를 기반으로 matchHistory 컬렉션에 게임 기록(시작/종료 시간, 양 플레이어 UID, 획득 점수, 게임 모드)을 업로드.
- **유저 전적 업데이트**:
  - 게임 종료 시 users 컬렉션 내의 gamesPlayed 횟수를 +1 증가시키고, 누적 점수나 승률 통계 데이터를 갱신.
