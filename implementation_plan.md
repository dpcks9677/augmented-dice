# 강화 증강(주사위 변형) 구현 계획

게임 내 증강 중 주사위의 형태, 색상, 눈금이 변하는 특수한 증강 7종에 대한 3D 렌더링 및 물리 엔진 연동 계획입니다.

## User Review Required
> [!IMPORTANT]
> 사용자의 피드백을 반영하여 계획을 수정하였습니다. 아래의 내용이 맞는지 다시 한 번 확인 부탁드립니다.

## Proposed Changes

### `src/DiceEngine.js`
*   **[MODIFY] 주사위 재질 생성 로직 개편**
    *   기존 `createDiceMaterials`를 `getMaterialForDie(config)`와 같은 팩토리 형태로 변경하여 주사위별 색상, 눈금(텍스트 또는 점)을 동적으로 캔버스에 그려 텍스처로 생성할 수 있게 합니다.
    *   **커스텀 렌더링 지원**: 눈금 대신 텍스트("+2", "-1", "💀")나 숫자(8면체용)를 렌더링하는 로직을 추가합니다.
    *   **특수 주사위 색상 적용**: 각 증강에 맞춰 다음 색상을 주사위 모델에 적용합니다.
        *   황금 주사위: "Fortuna Gold" 색상
        *   세븐스 다이스: "터키석" 색
        *   커플 주사위: `#ff2c97`
        *   프로모션 주사위: `#666A73`
        *   이상한 주사위: `#754581`
        *   8면 주사위: `#002F5E`
        *   묵직한 주사위: `#C42A12`
*   **[MODIFY] `roll` 파라미터 변경**
    *   `roll(count)` 대신 `roll(diceConfigs)` 배열을 받아, 각 주사위 모델(Box 또는 Octahedron), 물리 충돌체, 재질을 개별 적용합니다.
*   **[MODIFY] 8면 주사위(Octahedron)의 실제 물리 모델 추가**
    *   CANNON.js의 `ConvexPolyhedron`을 활용하여 8면체의 정점(Vertices)과 면(Faces) 데이터를 기반으로 실제 물리 엔진 상에서 8면체로 굴러가고 상호작용하도록 구현합니다.
*   **[MODIFY] `calculateDieValue` 로직 개선 (현행 유지 및 확장)**
    *   기존의 `calculateDieValue` 로직 구조를 최대한 유지하되, 8면체의 경우 해당 폴리곤의 면 노멀(Normal) 벡터들을 기반으로 가장 위(Y축)를 향하고 있는 면을 식별하는 알고리즘을 추가합니다.
*   **[MODIFY] 8면체 전용 호버링(Hover) 테두리 추가**
    *   마우스 오버 시 나타나는 노란색 테두리를 8면체 주사위일 경우 둥근 사각형이 아닌, 8면체(마름모 또는 육각형 형태의 투영 모양)에 맞는 커스텀 테두리로 렌더링합니다. (색상과 선 굵기는 기존과 동일하게 유지)

### `src/main.js`
*   **[MODIFY] 주사위 설정(Config) 주입**
    *   플레이어의 현재 활성화된 증강(activeMutations)을 확인하여, `engine.roll` 호출 시 일반 주사위뿐만 아니라 묵직한 주사위, 황금 주사위, 프로모션 주사위 등을 배열 형태로 구성하여 전달합니다.
*   **[MODIFY] 증강별 상태 추적 및 파괴 로직**
    *   **프로모션 주사위**: 라운드가 지날 때마다 눈금이 1씩 증가하는 로직(상태 관리) 구현.
    *   **이상한 주사위**: 족보를 확정(lockScore)할 때, 이상한 주사위의 눈금이 파괴(해골) 면이었다면 해당 주사위를 영구 제거하는 로직을 추가합니다.
*   **[MODIFY] 족보 표기 방식 변경**
    *   주사위 보너스로 얻은 추가 점수가 있을 경우, 족보 칸에 원래 점수와 추가 점수를 명확히 분리하여 `M +N` (예: 15 +3) 형태로 표기하도록 UI 업데이트를 진행합니다.

### `src/scoreEngine.js`
*   **[MODIFY] 특수 주사위 점수 계산 연동**
    *   황금 주사위, 커플 주사위, 이상한 주사위 등 기입 시 추가 점수를 얻거나 점수에 반영되는 보너스 로직을 족보 계산기 내부에 추가하고, 계산 결과로 원본 점수와 보너스 점수를 함께 반환하도록 구조를 개선합니다.

## Verification Plan

### Manual Verification
1.  로컬 핫시트 모드에서 각 특수 주사위 증강을 수동으로 활성화하여 굴려봅니다.
2.  8면 주사위가 물리적으로 정확히 상호작용하며 구르고, 마우스 호버 시 8면체 형태의 테두리가 나타나는지 확인합니다.
3.  이상한 주사위가 해골 면인 상태로 족보 확정 시 영구 파괴되는지 테스트합니다.
4.  족보 칸에 보너스 점수가 `M +N` 형태로 올바르게 렌더링되는지 확인합니다.
